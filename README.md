# ИИ-практикум — курс для работников

Статический сайт-курс (HTML/CSS/JS, без сборки). Работает на любом статическом хостинге, включая **бесплатный GitHub Pages**. Прогресс, тренажёр промптов и личная библиотека хранятся в браузере участника; результаты тестов уходят в Google-таблицу через Apps Script.

## Что нового в этой версии (v25)

**Содержание**
- 5 новых прикладных блоков: «Изображения, скриншоты и документы», «Таблицы и данные», «Большие документы и длинный контекст», «Письма, совещания и перевод», «Граница: что не стоит поручать ИИ». Всего 19 блоков.
- Цифры на стартовом экране считаются автоматически из данных курса.

**Интерактив и фишки**
- 🧪 **Песочница промптов** — мгновенная оценка промпта по 7 опорам (роль, задача, контекст, данные, формат, стиль, ограничения). Работает офлайн, без ключа.
- ▶ **Живой ответ модели** — по желанию: участник вводит свой API-ключ (OpenAI-совместимый), запрос идёт напрямую провайдеру. Ключ хранится только в браузере.
- 🎯 **Диагностика** — короткий пре-тест и рекомендация, с какого блока начать.
- 🃏 **Карточки глоссария** — флеш-карты с переворотом.
- 🔎 **Поиск** по темам, теории, вопросам и терминам.
- ★ **Личная библиотека промптов** — сохранение, экспорт и импорт своих шаблонов.
- 🎓 **Сертификат** — именной, печать в PDF или скачивание картинкой.
- 📊 **Радар результатов** по темам + 🎉 поздравление по завершении.
- 👋 Онбординг-тур, кнопка «Продолжить», мобильное меню, ссылка «к содержанию», улучшенная доступность.

**Технологии**
- 📲 **PWA** — устанавливается как приложение и работает офлайн (manifest + service worker).
- 📈 **Панель владельца** (`admin.html`) — сводная статистика из таблицы (опционально).

## Деплой на GitHub Pages (бесплатно)

1. Создайте репозиторий на GitHub (публичный — для бесплатного Pages).
2. Загрузите **все файлы из этой папки** в корень репозитория (или в папку `/docs`).
3. На GitHub: **Settings → Pages**.
4. **Source**: `Deploy from a branch`. Branch: `main`, папка `/ (root)` (или `/docs`, если положили туда). Save.
5. Через 1–2 минуты сайт будет доступен по адресу `https://<ваш-логин>.github.io/<репозиторий>/`.

Файл `.nojekyll` уже добавлен — он нужен, чтобы GitHub Pages отдавал все файлы как есть. PWA и service worker работают только по HTTPS (GitHub Pages всегда HTTPS) — на проде установка приложения и офлайн будут доступны автоматически.

### Обновление версии
Текущая сборка выпущена как `v27`. После любых правок поднимите номер версии:
- в `index.html` (`styles-v27.css`, `features-v27.css`, `script-v27.js`, `features-v27.js`),
- в `sw.js` и `sw-v27.js` (константа `CACHE = "ai-course-v…"`).

Иначе у части участников останется старая закэшированная версия.

## Песочница и ключ API (важно про CORS)

Живой ответ модели работает напрямую из браузера, поэтому провайдер должен разрешать запросы из браузера (CORS):
- **OpenRouter** — проще всего из браузера, рекомендуется по умолчанию.
- **OpenAI**, **Qwen (DashScope)** — могут блокировать браузерные запросы; тогда подойдёт только сервер-прокси.
- Любой **свой OpenAI-совместимый endpoint** — можно указать base URL и модель.

Без ключа песочница всё равно полезна: она оценивает структуру промпта. Ключ хранится в `localStorage` конкретного браузера и нигде не публикуется. На общем компьютере не используйте секретные рабочие ключи.

## Сбор результатов (Google Sheets)

Настройка таблицы и Apps Script описана в [google-sheets-setup.md](google-sheets-setup.md). Ссылка на веб-приложение указывается в `script.js` (константа `RESULTS_ENDPOINT`) и в `admin.html` (константа `ENDPOINT`).

### Панель владельца (опционально)
1. Обновите код таблицы содержимым `google-apps-script.js` (добавлено действие `stats`) и **передеплойте** веб-приложение (Deploy → Manage deployments → новая версия).
2. (Необязательно) задайте `OWNER_KEY` в Apps Script, чтобы статистику видели только вы.
3. Откройте `admin.html`, при необходимости введите ключ владельца → «Загрузить».

`admin.html` помечен `noindex` и не связан ссылками с курсом — давайте его адрес только себе.

|------|-----------|
| `index.html` | Точка входа курса |

| `admin.html` | Панель владельца (статистика) |
| `google-apps-script.js` | Серверная часть (вход, приём результатов, статистика) |
| `_headers`, `.nojekyll` | Служебные файлы хостинга |

## Локальный запуск

Из-за service worker нужен http-сервер (не открывайте `index.html` как `file://`):

```bash
npx serve .
# или
python3 -m http.server 8080
```

Затем откройте `http://localhost:8080`.
