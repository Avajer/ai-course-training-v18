# Сбор результатов курса в Google Sheets

## 1. Создайте таблицу

1. Откройте Google Sheets.
2. Создайте новую таблицу, например `Результаты курса ИИ`.
3. Откройте `Расширения -> Apps Script`.

## 2. Вставьте скрипт

1. Удалите стандартный код.
2. Вставьте содержимое файла `google-apps-script.js`.
3. Сохраните проект.

Скрипт сам создаст необходимые листы при первой отправке:

- `Пользователи` — регистрация участников, подразделение, технический хэш пароля, статус и последний вход.
- `Коды доступа` — закрытая регистрация: код, статус, ФИО, подразделение и дата использования.
- `Журнал доступа` — все попытки регистрации, входа и сброса пароля: дата, действие, ФИО, код, результат и сообщение.
- `Результаты теста` — общий балл, уровень, баллы по темам и ответы итоговой диагностики.
- `Открытые вопросы` — отдельная строка на каждый открытый ответ участника.
- `Практические задания` — отдельная строка на каждый ответ на практическое задание.
- `Мини-тесты` — результат проверки каждого учебного блока.

## 3. Опубликуйте веб-приложение

1. Нажмите `Deploy -> New deployment`.
2. Тип: `Web app`.
3. Execute as: `Me`.
4. Who has access: `Anyone with the link`.
5. Нажмите `Deploy`.
6. При первом запуске Google попросит разрешение на запись в таблицу. Разрешение нужно выдать владельцу таблицы.
7. Скопируйте ссылку веб-приложения.

При обновлении существующего проекта откройте `Deploy -> Manage deployments`, выберите действующее веб-приложение, создайте `New version` и нажмите `Deploy`. Простого сохранения кода недостаточно: старая версия продолжит отвечать сообщением `Неизвестное действие` или не примет пакетную отправку открытых и практических ответов.

После обновления проверьте ссылку вида:

```text
https://script.google.com/macros/s/ВАША_ССЫЛКА/exec?action=health&callback=cb
```

В ответе должны быть `version: "2026-07-01-v7-bulk-open-practice-results"` и возможности `submitOpenAnswers: true`, `submitPracticeAnswers: true`.

## 4. Подключите ссылку к курсу

В файле `script.js` ссылка уже указана в константе:

```js
const RESULTS_ENDPOINT = "https://script.google.com/macros/s/AKfycbzf89xEzwWUKKXtUMR9tBc4Lb34T2q9Ml5tJ371UOIYGpH1KLFtFML_hdIwpginJ3OV/exec";
```

Если вы создадите новое веб-приложение, замените значение этой константы на новую ссылку.

Регистрация закрытая: без кода доступа участник не сможет создать учетную запись. После регистрации вход выполняется по ФИО и паролю.

## 5. Как выдать доступ работникам

1. В Apps Script вставьте свежую версию `google-apps-script.js`.
2. Нажмите `Run` и запустите функцию `generateAccessCodes`.
3. Подтвердите разрешения, если Google попросит.
4. В таблице появится лист `Коды доступа`.
5. Выдайте работнику код из колонки `Код доступа` со статусом `active`.
6. Работник на сайте выбирает `Регистрация`, вводит ФИО, подразделение, код доступа и свой пароль.
7. После успешной регистрации код автоматически получит статус `used`.

Если хотите выдать именной код, заранее впишите ФИО работника в колонку `ФИО` на листе `Коды доступа`. Тогда этот код сможет использовать только участник с таким же ФИО.

Чтобы отключить невыданный код, замените его статус на `blocked`, `deleted` или `disabled`.

После 5 неудачных попыток регистрации с одним и тем же существующим кодом Apps Script автоматически меняет статус кода на `blocked`. Это помогает, если код переслали не тому человеку или кто-то подбирает ФИО под код.

## 6. Что отправляется

В `Результаты теста` попадает:

- дата отправки;
- ФИО;
- подразделение;
- процент;
- количество правильных ответов;
- уровень;
- баллы по темам;
- список пройденных блоков;
- ответы итогового теста.

В `Пользователи` попадает:

- дата регистрации;
- ФИО;
- подразделение;
- хэш пароля;
- статус;
- последний вход;
- комментарий владельца курса.

Пароль не сохраняется открытым текстом. В таблицу записывается только хэш, по которому Apps Script проверяет вход.

В `Журнал доступа` попадает:

- дата попытки;
- действие: `register`, `login`, `resetPassword`;
- ФИО;
- подразделение;
- код доступа, если он вводился;
- результат: `ok` или `error`;
- сообщение об ошибке или успешном действии.

В `Открытые вопросы` попадает:

- дата;
- ФИО;
- подразделение;
- блок;
- текст вопроса;
- ответ участника.

В `Практические задания` попадает:

- дата;
- ФИО;
- подразделение;
- блок;
- текст практического задания;
- ответ участника.

## 7. Как отключить непонятного пользователя

1. Откройте лист `Пользователи`.
2. Найдите нужного человека по ФИО.
3. В колонке `Статус` замените `active` на `blocked` или `deleted`.
4. Сохраните таблицу.

После этого пользователь не сможет войти с этим ФИО и паролем, а новые результаты от него не будут записываться.

Если отправка не прошла, участник может нажать `Скачать файл` и передать результат вручную.
